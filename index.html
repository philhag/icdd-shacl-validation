icdd-shacl-validation

This GitHub repository contains datasets, rules, and source code used in the paper "Semantic Rule Checking using SHACL for ICDD Information Containers: A Benchmarking Approach" by P. Hagedorn , P. Pauwels , and M. König to provide a benchmark of SHACL rules on ICDD information containers.

For the generation of ICDD containers the IIB.ICDD framework is used. IFC conversion is done using IFCtoLBD converter. XML conversion is done using msproj-xm-to-lbd converter.

Rules

The following rules are available as SHACL shapes in the Turtle format. Each rule is encoded once for the inferred dataset and once for the original dataset.

R1,V1: Each building element must be linked to (at least) one task.
R1,V2: Each building element must be linked to (at least) one task.
R2,V1: Tasks associated with the same building element may not overlap in time.
R2,V2: Tasks associated with the same building element may not overlap in time.
R3,V1: A task for load-bearing reinforced concrete walls should not take longer than the average coefficient of performance of 2.5h/m³ in terms of the ratio between volume per unit time.
R3,V2: A task for load-bearing reinforced concrete walls should not take longer than the average coefficient of performance of 2.5h/m³ in terms of the ratio between volume per unit time.
